# HRPortal Output Structure

The HRPortal output structure consists of several key components, each serving a specific purpose in the overall architecture. Below is an outline of the structure:

## 1. Contracts
- Description of the contracts created or utilized within the HRPortal.

## 2. metadata.json
- Contains metadata information for the output files, including schema version, timestamps, and other relevant details.

## 3. permissions.json
- Defines the permissions applicable to various user roles and access levels within the HRPortal.

## 4. Policies
- Outlines the policies governing user interactions, data access, and security measures within the HRPortal.

## 5. provisioning.log
- A log file that captures all provisioning activities and operations performed, useful for troubleshooting and auditing.

## 6. Reports
- Contains autogenerated reports from HRPortal, which provide insights and analytics on the HR processes.

## 7. site-info.json
- A JSON file that holds information specific to the sites created or managed through HRPortal, including settings and configurations.
